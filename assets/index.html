HTML

  • The head element describes information regarding the webpage. It helps browsers interpret the content correctly.
  • The body element contains the information that is visible to the user
  • The h elements represent text headings of varied sizes. It is best practice to have only one h1 and use the consecutive h2, h3, h4, h5, or h6 elements for other headings throughout the text block.
  • The ul element is defined as an unordered list
  • The li element is used to add each line of information to the list
  • The p element is used for blocks of text or paragraphs
  • The img element contains the information about images within the webpage. Information within the img element is not visible to the user
  • The footer element appears at the bottom of the webpage. It typically contains naviagation, copyright information, authors.

Git

  • git add: adds a file to the next commit
  • git commit: commits new content in a snapshot. This is important to maintaining version control incase there is a need to revert back to a previous version of code
  • git checkout: switches from the current working directory to another branch
  • git merge: merges a specific branch into the current one. This allows developers to integrate changes within a branches history
  • git push: passes local branch commits to the remote repo branch
  • git pull: fetches and merges commits from the remote branch
  • git status: shows modified files in the working directory that are ready for the next commit
